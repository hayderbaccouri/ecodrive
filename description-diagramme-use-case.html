EcoDrive

Description du diagramme de cas d'utilisation
Projet de fin d'études • Juin 2026

1. Présentation

Le diagramme de cas d'utilisation modélise les interactions fonctionnelles entre les trois types d'acteurs et le système EcoDrive. Il permet de visualiser l'ensemble des fonctionnalités offertes par la plateforme, organisées par périmètre d'accès.

2. Acteurs

ActeurSymboleDescriptionPérimètre
Visiteur bleu Utilisateur non connecté naviguant sur le site Consultation uniquement (lecture seule)
Client orange Utilisateur inscrit et connecté Hérite du visiteur + actions personnelles (réservation, profil)
Administrateur vert Gestionnaire de la plateforme Hérite du client + back-office complet

Les flèches d'héritage entre acteurs (visiteur → client → administrateur) indiquent que chaque acteur hérite des cas d'utilisation du précédent : un administrateur peut tout faire, un client peut tout ce que fait un visiteur, etc.

4. Relations « include »

Deux relations d'inclusion relient les cas d'utilisation :

  • Réserver un essai → « include » → Se connecter : un client doit être authentifié pour pouvoir réserver. Le système redirige vers la connexion si l'utilisateur n'est pas connecté.
  • Consulter mes réservations → « include » → Réserver un essai : le dashboard affiche les réservations existantes et permet d'en créer une nouvelle.

5. Synthèse

Le diagramme présente 19 cas d'utilisation répartis sur 3 acteurs avec une hiérarchie d'héritage. Il couvre l'intégralité du périmètre fonctionnel du projet :

  • Consultation (visiteur) : navigation, découverte, inscription
  • Action (client) : réservation, suivi, gestion de compte
  • Gestion (administrateur) : administration complète, statistiques, export

Cette organisation illustre une séparation claire des responsabilités et un gradient de privilèges progressif, adapté aux besoins de chaque profil d'utilisateur.
